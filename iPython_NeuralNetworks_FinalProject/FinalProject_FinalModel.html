Split Data Into Train and Test Sets¶

I chose to split the data into testing and training data set based on the date 1/1/2015 00:00:00. Any points before this time are put into the training set. Any points after (and including) are put into the testing or validation set. I chose to use this split because this way the data used to create the model (the training data) is completely separate from data used to validate it (the testing set). I did not want overlap because using the same points for training and testing can lead to a model looking better than it actually is (overfitting). In the project instructions it said to use data from 12/31/2014 00:00:00 as testing data, but this would cause the overlap. Below I create another dataset that I refer to as prediction or Kaggle data that starts at 12/31/2014 00:00:00 and is used to generate the prediction data required for the Kaggle competition.

Transform Data in Keras RNN Input Format¶

The function below is used to create the data required for Keras LSTM modeling. The data must have time for all variables and all times steps present for each y value. In other words, for a time step of 24, the first y value, the x data must contain the t-24, t-23, t-22, ..., t-2, and t-1 values of all 14 variables. The first y value for a time step of 24 is the 24th entry (this will be the time t value). This function uses the shift property of pandas dataframes to shift the entire input dataframe down by a given amount. A list is generated, each entry of which is a dataframe with a different shift value (between 1 and 24). This list is then turned into one dataframe (will be the x dataframe) using the pandas concatenate property. The names of the columns of this new dataframe are generated by concatenating the original variable names with the time shift (t-24, t-23, etc). The y variable is shifted up by the time step, making the entry equal to the time step the first. All NaN values are then dropped to remove any shifted dataframe entries that have NaNs in any column. The index values of the y output are then set to the indices of the x dataframe to ensure the output x and y dataframes have same indices.

In [22]:
def create_RNN_input(in_df, timestep, dropnan, output_var):
    
    var_names = list(in_df.columns.values)
    num_vars = len(var_names)

    x_df_list = []
    x_df_names = []

    # input sequence (t-n, ... t-1)
    for i in range(timestep, 0, -1):
        x_df_list.append(in_df.shift(i))
        x_df_names += [('%s(t-%d)' % (var_names[j], i)) for j in range(len(var_names))]

    # put it all together
    x_df = pd.concat(x_df_list, axis = 1)
    x_df.columns = x_df_names

    # forecast sequence (t, t+1, ... t+n)
    y_df = in_df[output_var].shift(-timestep)

    # drop rows with NaN values
    if dropnan:
        x_df.dropna(inplace = True)
        y_df.dropna(inplace = True)

    out_ndx = list(x_df.index.values)
    y_df.index = out_ndx
    
    return x_df, y_df

Common Model Evaluation and Prediction Functions¶

These functions are used during the evaluation of models and the creation of prediction data. I created these functions because the same analyses are performed after every model. Making common analysis functions that can be called after each model reduces the amount of repeatitive code present and makes the model analysis outputs easier to read.

This function creates a plot of a model's training losses and validation losses over epochs. This plot is used to analyze if overfitting is present in the model. If a model is overfit, the training loss will reduce over epoch, while the validation loss either stays the same or increases. This will cause a widening gap between the training and validation losses.

In [43]:
def plot_model_losses(model_hist_dict, model_desc):
    model_loss = model_hist_dict['loss']
    model_val_loss = model_hist_dict['val_loss']

    model_epochs = range(1, len(model_loss) + 1)
    
    # "bo" is for "blue dot"
    plt.plot(model_epochs, model_loss, 'bo', label = 'Training Loss')
    
    # b is for "solid blue line"
    plot_title = model_desc + ' Training and Validation Loss'
    plt.plot(model_epochs, model_val_loss, 'b', label = 'Validation Loss')
    plt.title(plot_title)
    plt.xlabel('Epochs')
    plt.ylabel('Loss')
    plt.legend()

    plt.show()

def evaluate_model(model, model_hist_dict, x_test_np, y_test_np, pred_x_test_np, pred_y_test_np, model_batch, 
                   model_num):
    
    score = model.evaluate(x_test_np, y_test_np, batch_size = model_batch)
    print('Model %d Test Loss Score: ' % model_num, score)
    
    pred_score = model.evaluate(pred_x_test_np, pred_y_test_np, batch_size = model_batch)

    plot_model_losses(model_hist_dict, 'Model %d' % model_num)
    
    return score, pred_score

This function calculates predictions for the test x data. Dataframes are created for the predictions, with the rest of the original test variable columns except the temperature column the same. The prediction data is then set to the new temperature column. The order of the variables (columns) in the new dataframe matches that of the normalized data. This new dataframe is then sent through the inverse normalization transform. This un-normalizes the prediction y data. Another dataframe is created with the original variable columns and the test y data as the temperature column. This dataframe is then sent through the inverse normalization transform as well. This will produce the truth y data. The RMSE and MAE are then calculated between the unnormalized prediction and truth data.

In [45]:
def create_model_predictions(model, x_test_np, norm_test_df, y_test_np):
    
    model_predictions = model.predict(x_test_np)
    
    num_test_pts = norm_test_df.shape[0]
    test_x_norm = norm_test_df.iloc[24:(num_test_pts + 1), :]
    date_time_x = pd.Series(test_x_norm.index.values)

    # invert scaling for forecast
    inv_x_y_norm = test_x_norm[:]
    inv_x_y_norm['temperature'] = model_predictions
    inv_x_y = min_max_scaler.inverse_transform(inv_x_y_norm)
    inv_x_y_df = pd.DataFrame(inv_x_y, columns = var_names, index = date_time_x)
    inv_yhat = inv_x_y_df['temperature']

    # invert scaling for actual
    inv_x_y_test_norm = test_x_norm[:]
    inv_x_y_test_norm['temperature'] = y_test_np
    inv_x_y_test = min_max_scaler.inverse_transform(inv_x_y_test_norm)
    inv_x_y_test_df = pd.DataFrame(inv_x_y_test, columns = var_names, index = date_time_x)
    inv_y = inv_x_y_test_df['temperature']

    # calculate RMSE
    rmse = np.sqrt(mean_squared_error(inv_y, inv_yhat))
    mae = mean_absolute_error(inv_y, inv_yhat)

    return inv_yhat, inv_y, date_time_x, rmse, mae

def get_loss_results(model_hist_dict):
    model_loss_np = np.array(model_hist_dict['loss'])
    model_val_loss_np = np.array(model_hist_dict['val_loss'])
    
    min_ndx = np.argmin(model_loss_np)
    min_loss = model_loss_np[min_ndx]
    min_val_loss = model_val_loss_np[min_ndx]
    
    last_loss = model_loss_np[-1]
    last_val_loss = model_val_loss_np[-1]
    
    return min_loss, min_val_loss, last_loss, last_val_loss

Final Model - 2 LSTM Layer With 50 Nodes Each, 20 Epochs, Batch Size 10¶

This is the best model that I found from multiple runs of the 33 models I tried. This model had consistently low losses and had the smallest variation in loss between multiple runs. It also was not overfit.

The model has 2 LSTM layers with 50 nodes each back to back, followed by one dense layer with 1 output. The 1 output of the dense layer corresponds to the 1 temperature output calculation. The activations used in the LSTM and dense layers is the default (None in Keras). This corresponds to a linear activation. Other activations were attempted but did not result in acceptable models.
